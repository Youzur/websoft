S09

One security issue that comes to mind is the Injections. If one does not have a secure web page, a hacker can easily enter malicious SQL sequences to gather information about customers or users accessing the website. Another security issue is IDOR. It is when a hacker can gain access to information by manipulating the URL sequencing when redirecting to a page. With this, one can download as many sensitive information of the users of that website. The cause of this is poor security URL management. XSS Cross-Site Scripting is a very unique was of manipulating a website with poorly coded sanitisation method. XSS is when a hacker enters his/her own JS script as an input, for example, in a input field of such poorly maintained website, and be able to create havoc, or steal information.

A good way to have a secure website when creating one is having a recommended password manager, for example when creating a Login functionality into the website. Another good practice is to have backups, in case an incident were to happen. This insures credibility for both the customers and the creators of the website. All the security issues that have been mentioned are important threats which should be looked into avoiding/patch from the start.

The TIL of this assignment is Web Security. Over the years, hackers have attacked many websites and gathered important data from customers. It is vital, as developers, to study and understand how a hacker thinks and learn to counter these malicious attacks.
